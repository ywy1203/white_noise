# 3D 粒子场景引擎 V1 — 产品需求与项目排期

> 基于 StudyCompanion 现有项目（Vue 3 + Pinia + Canvas 2D 视频渲染 + Web Audio API）  
> 将视频方案逐步替换为 Three.js 粒子渲染方案

---

## 1. 产品全景

### 1.1 项目定位

StudyCompanion 是一款白噪音陪伴型 App，当前使用 **7 个预录视频** 作为场景背景。  
V1 目标：**用 3D 粒子系统替代视频，让画面跟随音频实时律动，支持双端交互。**

### 1.2 当前技术栈

| 层级 | 技术 |
|------|------|
| 框架 | Vue 3 + TypeScript + Pinia |
| 构建 | Vite 6 |
| 当前渲染 | Canvas 2D（drawImage 逐帧绘制视频画面） |
| 目标渲染 | Three.js（Points 粒子系统） |
| 音频 | Web Audio API + COS 对象存储音频素材 |
| 部署 | Tencent COS + Capacitor（iOS/Android） |
| 兼容目标 | PC Chrome + iOS Safari + Android 浏览器 |

### 1.3 V1 范围边界

- **不做** 用户上传图片
- **不做** AI 指令解析
- **不做** 社区模板市场
- **做** 3-4 个预制粒子场景 + 音频律动 + 触摸交互 + 时间效果

## 2. 粒子场景清单（4 个预制场景）

### 2.1 🔥 篝火 · 冬夜（Campfire Night）

> **就是你正在开发的那一个。** 这个场景作为粒子引擎的"首发旗舰"。

| 项目 | 内容 |
|------|------|
| 参考图片方向 | 壁炉柴火堆、暗色泥土地面、温暖橙红光 |
| 对应视频原场景 | fireplace.mp4（替换） |
| 核心音效 | campfire（燃烧底噪）+ wind（风声）+ clock（时钟滴答） |
| 粒子结构 | 四层火焰 + 三层火星 + 双层烟雾 + 木柴堆 + 灰烬层 + 地面光影 |
| 音频联动 | 低频→火焰高度/亮度；中频→粒子抖动；高频→火星迸发 |
| 交互 | 触摸扰动火苗、单击迸发火星、长按火焰放大 |
| 色调 | `#ff8c42` 暖橙 |

### 2.2 🌊 海浪 · 沙滩（Ocean Waves）

> 粒子模拟海浪拍打沙滩，水面起伏 + 泡沫消散 + 海风拂面

| 项目 | 内容 |
|------|------|
| 参考图片方向 | 海滩俯瞰/平视、海浪线、浅蓝到深蓝渐变 |
| 对应视频原场景 | ocean.mp4（替换） |
| 核心音效 | waves（海浪）+ seagulls（海鸥）+ wind（风声）+ underwater（水下低频） |
| 粒子结构 | 四层粒子体系（由下到上）：海底深蓝粒子（静止基底）→ 中层海水粒子（横向来回涌动）→ 表层浪花白沫粒子（破碎/消散）→ 海面反光粒子（闪烁高光）+ 空中水雾粒子（随风飘散） |
| 音频联动 | 低频（海浪节奏）→ 水面起伏幅度/速度；中频（海风声）→ 水雾飘散速度；高频（浪花拍碎）→ 白沫迸溅/反光闪烁 |
| 交互 | 触摸水面→涟漪扩散；双击→浪花炸开；长按→潮汐加速 |
| 色调 | `#3a7bd5` 海蓝 → `#87CEEB` 浅蓝 |

**特殊机制 — 潮汐节奏：**
- 内置一个超低频振荡器（周期约 12s），模拟潮汐涨落的自然韵律
- 涨潮时粒子整体上涌、密度增加；退潮时粒子回落、露出湿沙粒子层
- 潮汐周期与音频低频波形叠加，互相调制

### 2.3 🌧️ 雨窗 · 咖啡馆（Rainy Window）

> 雨滴在玻璃窗上流淌 + 窗外城市灯光朦胧 + 呼吸出来的雾气

| 项目 | 内容 |
|------|------|
| 参考图片方向 | 咖啡馆玻璃窗、窗外雨景、霓虹灯光倒影、水珠滑落 |
| 对应视频原场景 | cafe_rain.mp4（替换） |
| 核心音效 | rain-on-window（雨打窗）+ cafe（咖啡馆底噪）+ typewriter（打字机）+ thunder（雷声） |
| 粒子结构 | 三层粒子体系：窗外雨丝粒子（倾斜下落，速度分层）→ 玻璃水珠粒子（沿窗面滑落，汇聚/滴落）→ 玻璃雾气粒子（呼吸凝雾/消散）+ 窗外城市光斑粒子（模糊虚化光点） |
| 音频联动 | 低频（雷声/雨势底噪）→ 雨量密度/倾斜角度；中频（咖啡馆人声底噪）→ 雾气凝结速度；高频（雨滴击打峰值）→ 单个水珠迸裂/滑落加速 |
| 交互 | 触摸玻璃→划开水雾，露出清晰窗外画面（雾粒子消散）；双击→模拟闪电白光闪烁；长按→呼出的热气在玻璃上凝成更大雾面 |
| 色调 | `#5b7b9a` 冷灰蓝 → 窗外暖色光斑 `#ffd700` |

**特殊机制 — 触摸划雾：**
- 手指/鼠标划过的地方，雾气粒子瞬间消散
- 露出下方的雨丝和城市光斑
- 抬手后雾气缓慢重新凝结（约 5s 恢复）
- 这个交互的"爽感"非常高，适合作为场景亮点

### 2.4 🍃 森林 · 萤火（Forest Fireflies）

> 月光下的森林空地，萤火虫缓缓飞舞 + 远处流水声

| 项目 | 内容 |
|------|------|
| 参考图片方向 | 夜间森林、月光穿透树冠、地面苔藓、萤火虫光点 |
| 对应视频原场景 | forest.mp4（替换） |
| 核心音效 | crickets（蟋蟀）+ owl（猫头鹰）+ wind-in-trees（林风）+ river（溪流） |
| 粒子结构 | 三层粒子体系：月光粒子（从树冠缝隙投射的丁达尔光柱/光斑，缓慢移动）→ 萤火虫粒子（金色光点，随机游走/闪烁/聚集）→ 地面苔藓微光粒子（极弱绿蓝自发光，静止）+ 飘浮花粉/尘埃粒子（极慢浮动） |
| 音频联动 | 低频（溪流/风声）→ 光柱缓慢摆动幅度；中频（蟋蟀鸣叫）→ 萤火虫闪烁频率/亮度；高频（夜间零碎声响）→ 个别萤火虫突然变亮/加速飞行 |
| 交互 | 触摸→萤火虫向手指聚集（被吸引）；双击→所有萤火虫同时闪烁一次；长按→萤火虫从手指位置向外扩散飞走 |
| 色调 | `#4a6fa5` 森林夜蓝 → `#2d5a27` 深绿 → `#ffd700` 萤火虫金 |

**特殊机制 — 萤火虫集群 AI（伪 AI，纯算法）：**
- 每只萤火虫有独立的 `phase` 偏移（0~2π）
- 集群行为使用简单的 Boid 算法变体：
  - 吸引力：趋向场景中心 + 趋向手指位置（触摸时）
  - 排斥力：彼此间距过近时推开，避免重叠
  - 随机力：每帧叠加微小随机偏移，产生"飘忽"感
- 闪烁模式：亮度 = `sin(time * freq + phase)`，每只频率略有差异

## 3. 时间效果设计

### 3.1 番茄钟律动（Pomodoro Rhythm）

> 与现有的 TimerPanel 番茄钟联动

| 状态 | 粒子效果 |
|------|---------|
| **专注中（25min）** | 场景稳定/平静，粒子运动幅度 70%，色调偏冷/中性，无额外特效 |
| **专注结束前 30s** | 粒子缓慢加速，亮度逐渐提高，提示用户即将休息 |
| **休息中（5min）** | 场景活跃，粒子运动幅度 120%，色调偏暖，额外生成庆祝粒子（如萤火虫飞舞加快/火星增多） |
| **休息结束前 10s** | 粒子逐渐恢复平静，亮度降低 |
| **连续完成 4 个番茄钟** | 触发"里程碑特效"：所有粒子一次性大爆发（类似篝火爆燃），持续 3s 后恢复正常 |

**实现方式：**
- TimerPanel 已有 `timerState`（mode/running/elapsed/duration）
- 粒子引擎监听番茄钟状态变化，通过 `t = (elapsed / duration)` 归一化时间插值控制参数
- 参数过渡使用 `lerp()` 平滑，避免突变

### 3.2 昼夜缓慢循环（Day/Night Cycle）

> 粒子场景的色调/亮度随真实时间缓慢变化

| 时段 | 色温 | 亮度 | 氛围 |
|------|------|------|------|
| 05:00-07:00 黎明 | `#ffe4b5` → `#ffd700` | 低→中 | 光线从地平线漫入，粒子缓慢苏醒 |
| 07:00-17:00 白天 | `#ffffff` | 高 | 场景明亮，粒子活跃度最高 |
| 17:00-19:00 黄昏 | `#ff8c42` → `#dc143c` | 中→低 | 暖色调，粒子趋于安静 |
| 19:00-05:00 夜晚 | `#1a1a2e` → `#16213e` | 低 | 深色，萤火虫/篝火成为主光源 |

**实现方式：**
- 用 `Date.now()` 获取当前小时，映射到 0-1 的昼夜曲线
- 通过 **环境光颜色 lerp** + **粒子基础亮度乘数** 实现
- 用户可开启/关闭此功能（默认开启）
- 循环周期 24h，非加速——用户几乎感知不到变化，但待久了会察觉

### 3.3 累积效应（Evolving Atmosphere）

> 用户听得越久，粒子世界逐渐"生长"

| 时间 | 变化 |
|------|------|
| 0-5 min | 标准粒子数量，常规运动 |
| 5-15 min | 粒子数量逐渐增加至 120%，新增稀薄飘浮粒子 |
| 15-30 min | 粒子摆动幅度略微增大，出现极低频呼吸效果 |
| 30-60 min | 粒子密度达到 150%，新增一层极淡的背景光晕粒子 |
| 60+ min | 维持稳定，不再增长，防止性能耗尽 |

**实现方式：**
- 用 `setInterval` 每 5 分钟检查一次，递增 `densityLevel`
- 每次增加时用 30s 过渡，不会让用户察觉突变
- 用户切换场景时重置为基准值

### 3.4 随机邂逅（Random Moments）

> 每隔一段时间触发一个小惊喜，增强陪伴感

| 场景 | 触发间隔 | 事件 |
|:----:|:--------:|------|
| 🔥 篝火 | 15-25min | 柴堆塌陷，火焰短暂暴涨，大量火星飞溅，持续 8s |
| 🌊 海浪 | 20-30min | 一阵强风吹过，浪花骤然加大，水雾弥漫，持续 10s |
| 🌧️ 雨窗 | 25-35min | 远处一道闪电，玻璃被照亮 2s，雨势短暂加大，持续 12s |
| 🍃 森林 | 15-20min | 一阵风穿过树冠，大量萤火虫同时亮起，花粉飘散，持续 10s |

**实现方式：**
- 每个场景内置一个 `randomEventTimer`，初始值在范围内随机
- 倒计时归零时触发特效参数组，持续指定秒数后复原
- `randomEventTimer` 重置为新的随机值

## 4. 场景切换与架构设计

### 4.1 场景定义格式（config-driven）

```typescript
// white-noise/src/config/particleScenes.ts
export interface ParticleSceneDef {
  id: string
  icon: string
  name: string
  /** 参考图片路径（用于粒子采样） */
  refImage: string
  /** 场景色调 */
  tone: string
  /** 粒子系统参数 */
  particles: {
    count: number           // 粒子总数
    distribution: 'color_sample' | 'procedural'
    layers: ParticleLayer[]
  }
  /** 音频关联行为 */
  audioBindings: {
    lowFreq: string         // 低频控制参数名
    midFreq: string
    highFreq: string
  }
  /** 交互行为 */
  interaction: {
    touch: 'ripple' | 'attract' | 'repel' | 'swipe'
    click: 'burst' | 'flash' | 'none'
    longPress: 'amplify' | 'gather' | 'scatter'
  }
  /** 特殊时间效果 */
  timeEffects: {
    pomodoro: boolean
    dayNight: boolean
    accumulation: boolean
    randomEvents: RandomEventDef[]
  }
}
```

### 4.2 场景切换流程

```
用户点击场景图标
  ↓
ParticleEngine.stop()         → 当前场景粒子渐隐（500ms fadeOut）
  ↓
ParticleEngine.load(config)   → 加载新场景配置
ParticleEngine.start()        → 新场景粒子渐显（500ms fadeIn）
  ↓
AudioEngine.applyMix(idx)     → 切换音频混合（已有逻辑，不改）
  ↓
TimerEffect.reset()           → 时间效果重置（昼夜循环保留）
  ↓
RandomEventTimer.reset()      → 随机邂逅重新计时
```

### 4.3 与现有架构的集成

```
App.vue
  ├── VideoCanvas.vue          → 保留（切换为三场景共用，向后兼容）
  ├── ParticleCanvas.vue       ← 新增（粒子场景渲染容器）
  ├── CtrlBar.vue              → 场景选择器增加粒子场景图标
  ├── SoundPanel.vue           → 已有逻辑，音频不受影响
  ├── TimerPanel.vue           → 新增→订阅粒子引擎的番茄钟状态
  └── sceneStore.ts            → 扩展场景列表，混合 video + particle 场景
```

不需要替换原本的 7 个视频场景，而是 **新增 4 个粒子场景**，用 `sceneType: 'video' | 'particle'` 区分。用户场景选择器里，视频场景和粒子场景并列显示。这样可以平滑过渡，视频场景逐步淘汰。

## 7. 技术风险与对策

| 风险 | 等级 | 对策 |
|------|:----:|------|
| Three.js 包体积 | 中 | 使用 ES module 按需引入（仅 Points + 基础模块，不引入整个库） |
| iOS Safari WebGL 半透明渲染 | 高 | 开发期用真机测试 blend mode，备选方案：使用 `transparent: false` + 预乘 alpha |
| 粒子数量过多导致移动端发热 | 高 | 动态降级：检测 `devicePixelRatio` + 屏幕分辨率 + 帧率，FPS < 25 自动减半粒子 |
| 与现有视频系统冲突 | 低 | sceneType 字段隔离，视频粒子互不干扰 |
| 白噪音音频无高频分量（如 brown-noise） | 低 | 模拟高频分量：对低频波形做微分/高速滤波，生成"伪高频"驱动火星特效 |
